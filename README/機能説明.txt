.Netフレームワーク windowsフォームでSQL,C#を使用してローカル環境で動作するスケジュール,家計簿管理アプリを作成しました。
特にこだわったのはデータ設計と操作性です。細かいところまで全て一人で考えて作成しました。
*機能説明*
ユーザーは任意のデータベースにパスワード,タイトル名を用いてログインまたはサーバーを作成します。
そして予定、家計簿を管理することができます。
接続文字列が自分のPC内のSQLサーバーに設定しているので、
Common.csのedittime_information.Title_Data_base_connect_codeをユーザーの使用するSQLサーバーの接続文字列に変更すれば
動作します。
*動機*
自分は昔から勉強の予定をすべて紙に書いていました。このやり方では紙を無駄に消費してしまいますし、効率が悪いと常々感じていました。
そこで、SQLの勉強を兼ねてスケジュール管理が出来るアプリを作ることを決めました。
SQLはクエリのみではなく正規化やデータの関連などについてもまた一から学習するきっかけとなったので有意義な経験となりました。
